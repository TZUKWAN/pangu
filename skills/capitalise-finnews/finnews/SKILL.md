# FinNews Skill

金融新闻数据查询与 Markdown 简报生成。专注一件事：**用最可靠的数据源拿到今日金融事实，输出标准结构的 Markdown 简报**。

## 核心职责

1. 从多数据源拉取今日金融数据（行情 + 新闻）
2. 整合成标准 Markdown 结构
3. 不脑补、不编造——一切以 API 返回为准
4. **日期必须准确**——以北京时间为准，不显示错误日期

## 数据源选择

### 路由优先级（第一原则）

| 用户请求 | 首选数据源 | 降级方案 |
|---|---|---|
| **宽问题**："今天财经大事"、"市场要闻" | 东方财富行情API + wallstreetcn快讯 | 新浪财经 |
| **美股个股**："AAPL新闻"、"特斯拉怎么了" | Yahoo Finance `quoteSummary?modules=news` | NewsAPI 搜索 |
| **A股个股**："茅台新闻"、"000001公告" | 东方财富公告API | — |
| **美股指数**："美股今天怎么样" | Yahoo Finance | NewsAPI |
| **宏观主题**："美联储"、"加息"、"CPI" | NewsAPI | Yahoo Finance搜索 |
| **大宗商品**："黄金"、"原油" | Yahoo Finance | NewsAPI |
| **加密货币**："比特币" | Yahoo Finance | NewsAPI |
| **财报季**："最近财报" | NewsAPI | Yahoo Finance |
| **A股板块/公告** | 东方财富 | — |

### 关键端点

**东方财富（中国大陆可用，免Key）**：
- 行情：`https://push2.eastmoney.com/api/qt/stock/get?secid=1.XXXXXX&fields=f43,f44,f45,f46,f47,f170`（上证指数1.000001，深证成指0.399001，创业板指1.399006，科创500.399673）
- 板块：`https://push2.eastmoney.com/api/qt/clist/get?pn=1&pz=15&po=1&np=1&fltt=2&invt=2&fid=f3&fs=m:90+t:1&fields=f2,f3,f12,f14`
- 公告：`https://np-anotice-stock.eastmoney.com/api/security/ann?page_size=20&page_index=1&ann_type=A,SHA,SWA&client_source=web`
- 搜索：`https://searchapi.eastmoney.com/api/suggest/get?input=公司名&type=14`

**Yahoo Finance（需海外网络，免Key）**：
- Base：`https://query1.finance.yahoo.com` 或 `https://query2.finance.yahoo.com`
- 个股新闻：`/v10/finance/quoteSummary/{symbol}?modules=news`
- 搜索：`/v1/finance/search?quotesCount=0&newsCount=20&q={keyword}`
- 常见symbol：`AAPL`, `TSLA`, `NVDA`, `^GSPC`(标普500), `^DJI`(道指), `^IXIC`(纳指), `GC=F`(黄金), `CL=F`(原油), `BTC-USD`(比特币)

**NewsAPI（全球可用，需Key）**：
- Base：`https://newsapi.org`
- 头条：`/v2/top-headlines?category=business&pageSize=30&apiKey={KEY}`
- 搜索：`/v2/everything?q={query}&from={YYYY-MM-DD}&sortBy=publishedAt&pageSize=30&apiKey={KEY}`
- 注册：https://newsapi.org/register（免费，100 req/天）

**web-access**（浏览器CDP，兜底方案）：
- wallstreetcn：`https://wallstreetcn.com/live/global`（全球快讯）
- 新浪财经：`https://finance.sina.com.cn/`（A股热点）

## 标准输出格式

**finnews 的唯一输出产物是 Markdown 文件**， 必须严格遵循以下结构

```markdown
# FinNews 金融热点简报 · YYYY-MM-DD

## 今日看点
1. **<标题>**：<一句话解读>。影响：<对普通人意味着什么>。
2. ...
（3-5条，每条包含"发生了什么"+"意味着什么"）

## A股异动
### 领涨板块
| 排名 | 板块 | 涨跌幅 |
|------|------|--------|
| 1 | 存储芯片 | +5.8% |
| 2 | 半导体 | +4.2% |
（3-5个，从东方财富板块API获取）

### 领跌板块
| 排名 | 板块 | 涨跌幅 |
|------|------|--------|
| 1 | 黄金 | -2.3% |
（2-3个）

### 异动解读
<一段话解读今日A股异动原因，200字以内>

## 全球要闻
### 美股指数
| 指数 | 表现 |
|------|------|
| 道琼斯 | 44,850，+0.3% |
| 纳斯达克 | 26,400，+1.2% |
| 标普500 | 7,490，+0.6% |
（从Yahoo Finance获取）

### 全球大事件
1. **<标题>**：<2-3句话摘要>
2. ...
（3条最重要的全球事件）

### 大宗商品
| 品种 | 价格 | 涨跌 |
|------|------|------|
| 现货黄金 | 4,480美元/盎司 | -1.2% |
| WTI原油 | 93.50美元/桶 | +0.8% |
（黄金+原油，2项）

### 一句话看懂
<一段话总结全球市场今日表现，100字以内>

## 概念科普
### <概念名称1>
**解释**：<用大白话解释，50字以内>
**为什么重要**：<跟今日新闻的关联，50字以内>

### <概念名称2>
**解释**：...
**为什么重要**：...
（3个今日出现的新概念/术语，用通俗语言解释）

## 资金动向
| 方向 | 标的 | 数据 |
|------|------|------|
| 北向资金 | 净流入 | 约45亿元 |
| 南向资金 | 净流入 | 约32亿港元 |
| <热门标的> | <涨跌幅> | <具体数据> |
（2-3条）

## 财经日历
- <今日/明日重要经济数据或事件>
- <今日/明日重要经济数据或事件>
（2-3条，用 `- ` 开头）

## 今日总结
<一段话总结今日市场+明日关注点，150字以内>

数据更新于 YYYY-MM-DD HH:MM
```

### 格式关键点

- **今日看点**：每条用 `1. **标题**：解读。影响：解读。` 格式
- **A股异动**：领涨/领跌用 Markdown 表格，异动解读用纯文本段落
- **全球要闻**：美股指数和大宗商品用表格，全球大事件用编号列表
- **概念科普**：每个概念用 `### 名称` + `**解释**：` + `**为什么重要**：` 三段式
- **资金动向**：用 Markdown 表格
- **财经日历**：用 `- ` 开头的列表
- **今日总结**：纯文本段落
- **时间转人话**：不要显示原始时间戳。24小时内显示"今天 XX:XX"，48小时内显示"昨天"，更早显示"5月27日"
- **中英文标题**：英文标题必须配中文要点概括
- **日期准确性**：使用北京时间，确保日期与实际一致

## 工作流

### 默认路径（宽问题）

1. 东方财富行情API → 获取A股主要指数和板块数据
2. wallstreetcn快讯（web-access CDP） → 获取全球财经快讯
3. 如有Yahoo Finance可达 → 获取美股指数和大宗商品数据
4. 整合为标准 Markdown 格式
5. 保存到 **************

### 个股查询路径

1. 识别股票代码（美股/A股/港股）
2. 根据路由表选择数据源
3. 拉取该股票/公司的最新新闻、公告、行情
4. 整理为个股简报格式
5. 展示给用户

### 主题查询路径

1. 识别主题（美联储/黄金/财报等）
2. 根据路由表选择数据源
3. 拉取相关关键词新闻
4. 整理为主题简报
5. 展示给用户

## 常见错误处理

| 错误 | 处理 |
|---|---|
| 东方财富API返回异常 | 降级到新浪财经或wallstreetcn |
| Yahoo Finance返回封锁页面 | 用NewsAPI替代（需key），或降级到wallstreetcn |
| NewsAPI限流（100 req/天） | 降级到东方财富或等次日重置 |
| 所有API失败 | 诚实告知用户，建议稍后重试 |
| 英文新闻无中文摘要 | 只展示标题+来源，不脑补内容 |
| 数据源返回为空 | 扩大搜索范围或换数据源 |

## 不要做

- **不要编造新闻内容**——永远以 API 返回为准，没有摘要时只展示标题+来源
- **不要暴露基础设施细节**——不要在输出中展示 API 路径、参数名、Unix 时间戳等技术术语
- **不要高频轮询**——Yahoo Finance 串行+间隔300ms，NewsAPI 注意日限流
- **不要把"今天市场怎么样"路由到单个个股**——先走市场概览聚合
